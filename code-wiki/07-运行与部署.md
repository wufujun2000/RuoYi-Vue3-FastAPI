# 运行与部署

## 1. 运行前准备

### 1.1 基础环境

- Node.js >= 18
- Python >= 3.10
- Redis >= 6.2
- MySQL >= 5.7 或 PostgreSQL >= 14
- 可选：Docker / Docker Compose

### 1.2 仓库结构要求

以下说明默认你在仓库根目录执行：

```bash
cd /workspace
```

## 2. 本地开发启动

## 2.1 启动后端

```bash
cd /workspace/ruoyi-fastapi-backend
pip3 install -r requirements.txt
python3 app.py --env=dev
```

如果使用 PostgreSQL：

```bash
cd /workspace/ruoyi-fastapi-backend
pip3 install -r requirements-pg.txt
python3 app.py --env=dev
```

### 2.1.1 后端运行前需要准备

- 配置 `.env.dev`
- 准备数据库
- 执行初始化 SQL：
  - MySQL：`sql/ruoyi-fastapi.sql`
  - PostgreSQL：`sql/ruoyi-fastapi-pg.sql`
- 确保 Redis 可访问

> 后端目录中已存在 `alembic/` 与 `alembic.ini`，但首次启动时表结构由 `lifespan` 中的 `init_create_table()`（`Base.metadata.create_all`）完成。
> 如果计划使用 Alembic 做后续迁移，建议二次开发前先与维护者确认演进策略，避免 `create_all` 与迁移脚本冲突。

### 2.1.2 默认后端关键配置

- 主机：`0.0.0.0`
- 端口：`9099`
- 根路径：`/dev-api`

## 2.2 启动 Web 前端

```bash
cd /workspace/ruoyi-fastapi-frontend
npm install
npm run dev
```

### 2.2.1 Web 开发配置要点

- Vite 端口：`80`
- 代理前缀：`/dev-api`
- 代理目标：`http://127.0.0.1:9099`

因此本地访问地址通常为：

- Web：`http://localhost:80`
- 后端 Swagger：`http://localhost:9099/docs` 或经过根路径代理后的文档地址

## 2.3 启动移动端

```bash
cd /workspace/ruoyi-fastapi-app
pnpm install
pnpm dev:h5
```

如果要启动微信小程序：

```bash
cd /workspace/ruoyi-fastapi-app
pnpm dev:mp-weixin
```

### 2.3.1 App 默认配置要点

- 默认后端地址：`http://localhost:9099`
- 页面注册在 `src/pages.json`
- 多端脚本统一定义在 `package.json`
- `pnpm dev` / `pnpm build` 默认目标平台是 **微信小程序（mp-weixin）**，启动 H5 请显式使用 `pnpm dev:h5`

## 3. 默认账户

## 4. 文档与调试入口

## 7. 环境文件说明

后端目录中可见以下环境文件：

- `.env.dev`
- `.env.prod`
- `.env.dockermy`
- `.env.dockerpg`

这些环境文件主要控制：

- 数据库连接
- Redis 连接
- JWT 配置
- 日志配置
- 传输层加密配置
- 应用模式与文档开关

Web 前端和移动端也分别有各自的环境或配置文件：

- Web：`.env.development`、`.env.production`、`.env.staging`、`.env.docker`
- App：`src/config.js` 与构建脚本参数

## 8. 推荐启动顺序

本地联调建议：

1. 先启动数据库和 Redis
2. 再启动后端
3. 等待后端初始化数据库、缓存、调度器成功
4. 再启动 Web 或 App
5. 最后再运行测试

## 9. 常见运行问题排查

### 9.1 前端能打开但接口失败

优先检查：

- 后端是否已启动在 `9099`
- Web 代理是否正常生效
- token 是否失效
- 传输层加密配置是否前后端一致

### 9.2 登录失败

优先检查：

- 验证码开关和验证码缓存
- Redis 是否可用
- 账户是否被锁定
- 数据库中管理员账户是否存在

### 9.3 后端启动异常

优先检查：

- `.env.*` 是否配置正确
- 数据库是否已初始化
- Redis 是否可连通
- 依赖是否与数据库类型匹配

### 9.4 调度任务不执行

优先检查：

- 当前 worker 是否拿到应用锁
- Redis 锁与同步通道是否正常
- 数据库中的任务状态是否启用
- 任务函数路径是否可被动态导入

## 10. 生产部署关注点

- `docker-compose.*.yml` 默认没有完整的数据持久化策略，需要自行补充。
- 生产环境应重点审查 JWT 密钥、数据库口令、Redis 口令、传输层私钥。
- 如果使用多 worker，需重点关注调度器 leader 行为和 Redis 稳定性。
- 若关闭 Swagger / ReDoc，需要通过环境变量关闭文档入口。

## 11. 运行方式总结

| 场景 | 推荐方式 |
| --- | --- |
| 单人本地开发 | 手工分别启动后端、Web、App |
| 前后端联调 | 后端 + Web 同时启动 |
| 小程序调试 | 后端 + `pnpm dev:mp-weixin` |
| 快速体验整套系统 | Docker Compose |
| 回归验证 | Docker 测试环境 + pytest-playwright |
