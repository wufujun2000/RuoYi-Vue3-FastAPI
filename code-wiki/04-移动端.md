# 移动端

## 1. 移动端定位

`ruoyi-fastapi-app` 是基于 uni-app 的多端应用，主要承载：

- 登录 / 注册
- 首页
- 工作台
- 我的 / 个人信息 / 设置
- 常见帮助与协议页

与 Web 管理端相比，移动端功能更聚焦，结构更轻量。

## 2. 目录结构

```text
ruoyi-fastapi-app
├── src/
│   ├── api/         # 接口封装
│   ├── pages/       # 页面组件
│   ├── plugins/     # auth / modal / tab 等插件
│   ├── store/       # Pinia 状态管理
│   ├── utils/       # 请求、鉴权、存储、校验、加解密
│   ├── main.ts      # 入口
│   ├── pages.json   # 页面注册与 tabBar 配置
│   ├── permission.js
│   ├── config.js
│   └── manifest.json
├── vite.config.ts
├── tailwind.config.ts
└── package.json
```

## 3. 技术栈

- uni-app
- Vue 3
- Vite
- TypeScript
- Pinia
- TailwindCSS
- node-forge

## 4. 入口 `main.ts`

`src/main.ts` 的职责比较明确：

- 创建 `createSSRApp(App)`
- 注册 Pinia store
- 注册插件 `install(app)`
- 注入全局字典方法 `useDict`
- 加载权限拦截 `permission.js`

Web 端入口更偏“全局 UI 装配”，移动端入口更偏“运行环境和基础能力装配”。

## 5. 页面注册 `pages.json`

`pages.json` 决定了整个 App 的页面路由和 tabBar。

### 5.1 页面模块

- `pages/login`
- `pages/register`
- `pages/index`
- `pages/work/index`
- `pages/mine/index`
- `pages/mine/avatar/index`
- `pages/mine/info/index`
- `pages/mine/info/edit`
- `pages/mine/pwd/index`
- `pages/mine/setting/index`
- `pages/mine/help/index`
- `pages/mine/about/index`
- `pages/common/agreement/index`
- `pages/common/privacy/index`
- `pages/common/webview/index`
- `pages/common/textview/index`

### 5.2 TabBar

底部 tabBar 提供三个主入口：

- 首页
- 工作台
- 我的

这表明移动端采用固定导航结构，不像 Web 端那样依赖后端动态菜单。

## 6. 页面权限拦截

`src/permission.js` 通过 `uni.addInterceptor()` 拦截以下跳转方法：

- `navigateTo`
- `redirectTo`
- `reLaunch`
- `switchTab`

### 6.1 白名单页面

- `/pages/login`
- `/pages/register`
- `/pages/common/webview/index`
- `/pages/common/agreement/index`
- `/pages/common/privacy/index`

### 6.2 拦截逻辑

- 已登录用户访问登录页时，重定向回首页
- 未登录用户访问非白名单页时，强制跳转到登录页

这与 Web 的路由守卫思想一致，但实现方式改为 uni-app 拦截器。

## 7. 请求层 `utils/request.js`

移动端请求封装与 Web 端思路基本一致，但底层使用 `uni.request()`。

### 7.1 请求发送前

- 附加 JWT token
- 调用传输层请求加密逻辑
- 将 GET `params` 拼接到 URL

### 7.2 响应返回后

- 对加密响应解密
- 基于业务 `code` 做统一判断
- 401 时弹出确认框并执行登出
- 在密钥失效时自动清空缓存并重试一次
- 统一处理网络错误、超时错误和 HTTP 错误

## 8. App 基础配置

`src/config.js` 中定义了：

- 后端基础地址 `baseUrl`
- App 名称、版本、logo
- 官网地址
- 协议链接信息

默认 `baseUrl` 为 `http://localhost:9099`，这更适合本地或模拟器环境调试。实际部署时通常需要按运行目标改写。

## 9. 构建配置

`package.json` 中提供了大量 uni-app 多端脚本：

- `pnpm dev`：等价于 `uni -p mp-weixin`，**默认目标平台是微信小程序**
- `pnpm build`：等价于 `npm run build:mp-weixin`，构建产物面向微信小程序
- `pnpm dev:h5`：本地 H5 开发
- `pnpm dev:mp-weixin`：微信小程序开发
- `pnpm build:h5`：构建 H5
- `pnpm build:mp-weixin`：构建微信小程序
- `pnpm build:app`：构建 App
- 其他 `dev:mp-*` / `build:mp-*` / `dev:quickapp-*` 系列脚本覆盖支付宝、百度、抖音、QQ、快应用等多端

> 项目 `dev` / `build` 缺省指向 mp-weixin，意味着移动端的优先适配目标是**微信小程序**，H5 / App 是补充能力。
> 二次开发时如果以 H5 或 App 为主，建议在使用方文档里改写默认脚本，避免误执行。

`vite.config.ts` 的关键点：

- 使用 `@dcloudio/vite-plugin-uni`
- 使用 `weapp-tailwindcss` 插件
- 用自定义插件修补 `node-forge` 在小程序环境下的运行兼容性
- 使用动态导入的 `unplugin-auto-import`

## 10. 与 Web 端的结构差异

| 维度 | Web 管理端 | 移动端 |
| --- | --- | --- |
| 路由来源 | 后端菜单树 + 本地静态路由 | `pages.json` 固定声明 |
| 权限控制 | Vue Router 路由守卫 | uni-app 跳转拦截器 |
| 页面规模 | 大量业务管理页面 | 少量核心页面 |
| 组件体系 | Element Plus + 自定义后台组件 | uni-app 原生组件体系 |
| 构建目标 | 浏览器后台管理系统 | H5 / 小程序 / App 多端 |

## 11. 关键函数

### 11.1 `createApp()`

职责：

- 创建 uni-app Vue 应用实例
- 注入状态和插件
- 作为多端运行的应用工厂

### 11.2 页面跳转拦截器

职责：

- 在页面跳转前统一判断登录态
- 保证受保护页面不会在无 token 情况下被访问

### 11.3 `request()`

职责：

- 统一发起网络请求
- 处理 token、加密、错误提示、自动重试
- 为所有 `src/api/*` 接口提供基础请求能力

## 12. 移动端阅读建议

建议按以下顺序阅读：

1. `src/main.ts`
2. `src/pages.json`
3. `src/permission.js`
4. `src/config.js`
5. `src/utils/request.js`
6. `src/store/modules/*`
7. `src/pages/*` 与 `src/api/*` 对照阅读
