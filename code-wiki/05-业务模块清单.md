## 5. 关键实体与模型命名习惯

后端遵循较稳定的命名约定：

- `*_do.py`：数据库实体对象
- `*_vo.py`：接口视图对象 / 请求响应模型
- `*_dao.py`：数据库访问层
- `*_service.py`：业务服务层
- `*_controller.py`：API 控制器

这使得定位代码时可以快速按层次跳转。

## 6. 模块定位方法

如果要快速定位某个功能，推荐使用下列路径映射：

1. 先在 Web `src/views` 或 App `src/pages` 找页面
2. 再查看对应 `src/api` 调用了哪个后端接口
3. 在后端 `controller` 找路由
4. 顺着进入 `service -> dao -> entity`

例如：

- “用户管理”链路：`views/system/user/index.vue -> api/system/user.js -> module_admin/controller/user_controller.py -> service/user_service.py`
- “代码生成”链路：`views/tool/gen/index.vue -> api/tool/gen.js -> module_generator/controller/gen_controller.py -> service/gen_service.py`
- “AI 对话”链路：`views/ai/chat/index.vue -> api/ai/chat.js -> module_ai/controller/ai_chat_controller.py -> service/ai_chat_service.py`
